## 🟠 Introduction 🟠
* **'회원', '상품', '주문' 도메인이 있는 간단한 웹 애플리케이션 설계 & 개발**
<br><br><br>

## 🟢 Review 🟢
단방향, 양방향, One To Oe, Many To One 등을 고려하여 테이블 설계하는 방법을 학습할 수 있었습니다. 그리고 엔티티 설계시 N+1 문제를 해결하기 위해 지연로딩(LAZY)으로 설정해야 되는 것과 fetch join의 역할을 알게 되었고, JPA 뿐만 아니라 DI (필드주입, 생성자주입)에 대한 내용과 Junit을 사용하여 테스트코드 작성하는 방법까지 학습할 수 있었습니다.
